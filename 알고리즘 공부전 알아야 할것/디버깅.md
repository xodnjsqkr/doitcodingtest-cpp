## 코드의 논리 오류를 잡는 방법

논리 오류를 찾는 가장 좋은 방법은 <b>디버깅</b>  
논리 오류가 아닌 문법오류는 컴파일러가 컴파일 과정중에 찾아줌  
논리 오류는 코드가 사용자의 의도와 다르게 동작하는 것, 다양한 형태로 발생  
  
## 디버깅 하는 방법  
1. 코드에서 디버깅 하고자 하는 줄에 중단점(break point)를 성정한다. 여러줄 가능  
2. IDE/Code Editor에 있는 디버깅 기능을 실행하여 1줄식 코드를 실행하거나  다음 중단점 까지 실행 가능  
이 과정중에 특정 변수를 지정하여 자신의 의도한 대로 바뀌는지 확인한다.  
3. 변수값 이외에도 원하는 수식을 사용하여 논리 오류를 파악할 수 있다.

## 하기 쉬운 실수 4가지
1. 변수 초기화 오류  
2. 반복문에서 인덱스 범위 지정오류  
3. 잘못된 변수 사용  
4. 자료형 범위 오류   

의도치 않게 출력값이 음수이면 변수 범위 초과 의심부터
